# Разрешения

<details open>

<summary>Разрешения</summary>

**Раздел разрешений** служит для управления возможностями сотрудников в системе. Через этот раздел для каждой должности или сотрудника точно определяется, к какому модулю можно получить доступ и какие операции можно выполнять. Это обеспечивает безопасность системы и позволяет контролировать работу каждого пользователя в рамках его обязанностей.

<figure><img src="../../.gitbook/assets/image (471).png" alt=""><figcaption></figcaption></figure>

Осуществляется вход в раздел **HR → Разрешения** и через кнопку **"Создать"** создается новое разрешение (role/permission set).

В этом разделе все разрешения отображаются в виде таблицы (table).

**Столбцы таблицы:**

* **Название** – название разрешения (role)

<figure><img src="../../.gitbook/assets/image (480).png" alt=""><figcaption></figcaption></figure>



При создании нового разрешения настраиваются права в разрезе модулей системы. Каждый модуль открывается отдельно, управляются операции внутри него.

<div><figure><img src="../../.gitbook/assets/image (481).png" alt=""><figcaption></figcaption></figure> <figure><img src="../../.gitbook/assets/image (482).png" alt=""><figcaption></figcaption></figure></div>

Для каждого модуля доступны следующие основные операции:

* **Создание** – добавление новой информации
* **Просмотр** – просмотр данных
* **Изменение** – редактирование существующих данных
* **Удаление** – удаление данных
* **Экспорт** – выгрузка данных

Для каждой операции существует 3 варианта состояния:

* **Запрещено** – разрешение полностью отсутствует
* **Если ответственный** – может работать только с данными, относящимися к нему
* **Разрешено** – может работать со всеми данными

Кроме того, для полного включения или отключения каждого модуля имеется общий переключатель. Если модуль отключен, все разрешения внутри него автоматически аннулируются.

Разрешения работают **взаимосвязанно** друг с другом. Например, если сотруднику-продавцу предоставлено разрешение только на раздел **Продажи**, а разрешения раздела **Склад** ограничены:

* он не сможет просматривать товары
* не сможет видеть складские остатки
* в результате не сможет полноценно выполнять процесс продажи

По этой причине важно учитывать взаимосвязь между модулями при настройке разрешений. С практической точки зрения:

* для работы продаж должно быть как минимум разрешение на просмотр товаров и склада
* неправильно настроенные разрешения могут ограничить работу в системе

Через этот раздел можно точно управлять тем, кто что может делать в системе, стандартизировать роли и обеспечить безопасность на высоком уровне.
