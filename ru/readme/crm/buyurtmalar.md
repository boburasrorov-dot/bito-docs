# Заказы

<details>

<summary>Список заказов</summary>

При входе в раздел **CRM → Заказы** отображается список всех созданных в системе заказов.

<figure><img src="../../.gitbook/assets/image (342).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (343).png" alt=""><figcaption></figcaption></figure>

Через этот раздел можно создавать, управлять и контролировать заказы, поступившие от клиентов.

Заказ — это запрос клиента на продукт или услугу, который впоследствии преобразуется в продажу.

</details>

<details>

<summary>Создание заказа</summary>

Для создания нового заказа нажимается кнопка **Создать** и выполняются следующие шаги:

<figure><img src="../../.gitbook/assets/image (344).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (345).png" alt=""><figcaption></figcaption></figure>

Основная информация

• **Номер** — номер заказа\
• **Клиент** — выбирается клиент, делающий заказ\
• **Статус** — статус заказа (новый, в процессе и т.д.)\
• **Дата** — дата создания заказа\
• **Ответственное лицо** — сотрудник, ответственный за заказ\
• **Посредник** — если есть, выбирается посредник\
• **Дата доставки** — дата доставки продукта\
• **Адрес доставки** — адрес клиента\
• **Валюта** — тип валюты\
• **Примечание** — дополнительная информация\
• **Загрузить файл** — документы, относящиеся к заказу

Продукты

<figure><img src="../../.gitbook/assets/image (346).png" alt=""><figcaption></figcaption></figure>

• В заказ добавляются продукты или услуги\
• Для каждого продукта вводятся:\
  — количество\
  — цена\
  — склад

• Система автоматически рассчитывает:\
  — общее количество\
  — общую цену\
  — скидки и налоги



Дополнительные расходы

<figure><img src="../../.gitbook/assets/image (347).png" alt=""><figcaption></figcaption></figure>

* &#x20;Можно добавить дополнительные расходы к заказу\
  (например: доставка, плата за услуги и т.д.)
* Расходы влияют на общую цену

**Как это работает**

• Клиент размещает заказ\
• Заказ вносится в систему\
• Указываются продукты и цены\
• Заказ завершается\
• Заказ **переводится в продажу**

**Важный момент**

• После завершения заказа его можно преобразовать в документ продажи\
• Через это заказ переходит в реальную продажу и начинается расчет

</details>

<details>

<summary>Список заказов (канбан вид)</summary>

В разделе заказов можно также отслеживать заказы в виде Канбан (по колонкам).

Этот вид упрощает управление процессом заказов, группируя их по статусу и визуализируя их процесс.

<figure><img src="../../.gitbook/assets/image (349).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (350).png" alt=""><figcaption></figcaption></figure>

**Основные возможности**

• Просмотр заказов в колонках по статусам\
• Перемещение заказов из одной колонки в другую\
• Быстрое управление статусом каждого заказа\
• Визуальный контроль процесса

**Колонки Канбан**

• **Новые** — вновь созданные заказы\
• **В процессе** — заказы, находящиеся в работе\
• **Выполнено** — завершенные заказы\
• **Отменено** — отмененные заказы

После этапа **В процессе** пользователь может по желанию добавлять этапы и присваивать им названия

**Результат**

• Четкое видение процесса заказа\
• Контроль работы сотрудников\
• Быстрое выявление проблемных этапов\
• Упрощение управления заказами

</details>

<details>

<summary>Dashboard</summary>

В разделе CRM → Dashboard отображается общая статистика и аналитические данные по заказам.

<figure><img src="../../.gitbook/assets/image (351).png" alt=""><figcaption></figcaption></figure>

Через этот раздел владелец бизнеса может отслеживать статус заказов, динамику и эффективность в одном месте.

<figure><img src="../../.gitbook/assets/image (353).png" alt=""><figcaption></figcaption></figure>

В верхней части Dashboard отображаются основные показатели по заказам:

• **Новые заказы** — количество поступивших новых заказов\
• **Заказы в процессе** — заказы, находящиеся в выполнении\
• **Закрытые заказы** — успешно завершенные заказы\
• **Отменено** — отмененные заказы\
• **Общая сумма заказов** — общая стоимость всех заказов

Этот блок позволяет быстро оценить общее состояние бизнеса.



**Динамика заказов**

Этот график показывает рост или снижение заказов по времени.

<figure><img src="../../.gitbook/assets/image (354).png" alt=""><figcaption></figcaption></figure>

• **Текущий период** — результаты за текущий временной промежуток\
• **Прошлый период** — сравнение с предыдущим периодом

Через фильтр можно выбрать следующее:

• Сумма заказов\
• Количество заказов

Через это бизнес:

• видит рост или снижение продаж\
• сравнивает с предыдущим периодом\
• определяет тренд



**Воронка заказов**

<figure><img src="../../.gitbook/assets/image (355).png" alt=""><figcaption></figcaption></figure>

Показывает, сколько заказов находится на каждом этапе:

• Новые\
• В процессе\
• Выполнено

Кроме того:

• отображается доля каждого этапа (%)\
• показывается среднее время выполнения

Через это:

• можно определить, на каком этапе «скапливается» много заказов\
• можно оценить эффективность процесса



**Наиболее заказываемые продукты**

В этом блоке показываются продукты с наибольшими продажами.

<figure><img src="../../.gitbook/assets/image (356).png" alt=""><figcaption></figcaption></figure>

• название продукта\
• сколько раз был заказан\
• доля (%)

Через это бизнес:

• знает, какие продукты пользуются спросом\
• правильно управляет ассортиментом



**Клиенты с наибольшим количеством заказов**

<figure><img src="../../.gitbook/assets/image (357).png" alt=""><figcaption></figcaption></figure>

Здесь показываются наиболее активные клиенты:

• имя клиента\
• общая сумма заказов

Через это:

• можно определить наиболее важных клиентов\
• разработать стратегию индивидуальной работы с ними



**Сотрудники с наибольшими продажами**

<figure><img src="../../.gitbook/assets/image (358).png" alt=""><figcaption></figcaption></figure>

Этот блок показывает эффективность сотрудников:

• имя сотрудника\
• количество заказов\
• общая сумма\
• доля в общих продажах (%)

Через это:

• определяются наиболее сильные продавцы\
• оценивается эффективность команды
