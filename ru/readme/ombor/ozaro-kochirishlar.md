# Взаимные перемещения

<details>

<summary>Запросы на взаимное перемещение</summary>

Через раздел запросов на взаимное перемещение предварительно создается запрос (request) для перемещения товаров между организациями. Этот этап представляет процесс согласования и планирования перед фактическим перемещением.

Пользователь заходит в раздел **Склад → Запросы на взаимное перемещение** и через кнопку "Создать" создает новый запрос.

<figure><img src="../../.gitbook/assets/image (407).png" alt=""><figcaption></figcaption></figure>

Вводятся следующие данные:

* **Организация-отправитель** — организация, отправляющая товар
* **Статус** — статус запроса (например: Новый, Утвержден)
* **Номер CTR** — номер документа
* **Время** — время создания запроса
* **Время перемещения** — планируемая дата
* **Отправитель** — ответственный сотрудник
* **Причина** — причина перемещения
* **Загрузить файл** — прикрепление документа (необязательно)
* **Комментарий** — дополнительная информация

В нижней части добавляются товары:

* выбирается товар
* вводится необходимое количество
* общее количество рассчитывается автоматически

<figure><img src="../../.gitbook/assets/image (408).png" alt=""><figcaption></figcaption></figure>

Дополнительные возможности:

* **Импорт** — загрузка товаров через файл
* **Быстрый выбор** — быстрое добавление
* **Автозаполнение** — автоматический ввод количества

Через данный раздел движение товаров между организациями планируется заранее и контролируется.



</details>

<details>

<summary>Взаимные перемещения</summary>

Через раздел взаимных перемещений осуществляется реальное перемещение товаров между организациями. Этот документ обычно создается на основе запроса на взаимное перемещение.

Пользователь заходит в раздел **Склад → Взаимные перемещения** и нажимает кнопку "Создать".

<figure><img src="../../.gitbook/assets/image (409).png" alt=""><figcaption></figcaption></figure>

Вводятся следующие данные:

* **Организация-получатель** — организация, принимающая товар
* **Статус** — статус документа
* **Номер CT** — номер документа
* **Время** — время операции
* **Дата отправки** — дата отправления
* **Склад отправителя** — склад, с которого отпускается товар
* **Отправитель** — ответственный сотрудник
* **Отметить как долг** — отмечается, если товар передается в долг
* **Причина** — причина перемещения
* **Загрузить файл** — прикрепление документа
* **Комментарий** — дополнительный комментарий

В разделе товаров:

* выбирается товар
* показывается доступное количество на складе
* отображается также количество в организации-получателе
* вводится перемещаемое количество
* при необходимости выбирается срок годности

<figure><img src="../../.gitbook/assets/image (410).png" alt=""><figcaption></figcaption></figure>

Дополнительные возможности:

* **Импорт** — загрузка товаров
* **Быстрый выбор** — быстрое добавление
* **Автозаполнение** — заполнение на основе остатков

Через данный раздел:

* управляется движение товаров между организациями
* остатки автоматически обновляются с обеих сторон
* ведутся перемещения на основе долга
* упрощаются процессы логистики и учета

Например:\
Если одна организация отправляет товар в другой филиал, этот процесс оформляется через документ взаимного перемещения, и остатки товаров соответственно изменяются в обеих организациях.



</details>
