# Инвентаризация



### Инвентаризация

Через раздел инвентаризаций осуществляется процесс сопоставления остатков товаров на складе с фактическим состоянием и выявления расхождений. Данный процесс обеспечивает полный контроль над товарами на складе, а также позволяет выявлять ошибки в учете.

Пользователь заходит в раздел **Склад → Инвентаризации** и нажимает кнопку **"Создать"**.

#### **Основные данные**

При создании инвентаризации вводятся следующие данные:

<figure><img src="../../.gitbook/assets/image (361).png" alt=""><figcaption></figcaption></figure>

* **Ответственное лицо** — сотрудник, ответственный за инвентаризацию
* **Склад** — инвентаризируемый склад
* **Дата начала** — время начала процесса
* **Дата окончания** — время завершения процесса
* **Примечание** — дополнительная информация (необязательно)
* **Загрузить файл** — прикрепить документ (необязательно)

Дополнительные настройки:

* **Полная** — все товары на складе автоматически добавляются в инвентаризацию
* **Частичная** — в инвентаризацию добавляются только выбранные товары

При установке **"Создать с количеством на складе"** система формирует товары с текущим остатком.



#### **Товары**

Если выбрано **Частичная**, пользователь вручную выбирает инвентаризируемые товары в нижнем окне.

Если выбрано **Полная**, все товары автоматически добавляются в список.

По каждому товару:

**Количество на складе** — текущий остаток в системе\
**Подсчитанное количество** — фактически подсчитанное количество\
**Разница** — рассчитывается автоматически

<figure><img src="../../.gitbook/assets/image (364).png" alt=""><figcaption></figcaption></figure>

#### **Процесс**

После ввода всех данных пользователь:

**Сохранить** — создается документ инвентаризации и открывается для последующего подсчета

**Сохранить и подсчитать** — документ сохраняется и пользователь сразу переходит в окно подсчета

В процессе подсчета пользователь вводит фактическое количество товаров.
