# Склады

<details>

<summary>Склады</summary>

Через раздел «Склады» создаются и управляются места хранения товаров (склады) в системе. Этот раздел позволяет правильно организовать структуру складов и точно контролировать движение товаров.

Для создания склада пользователь переходит в раздел **Склад → Склады** из левого меню и нажимает кнопку «Создать».

<figure><img src="../../.gitbook/assets/image (395).png" alt=""><figcaption></figcaption></figure>

Раздел «Склады» служит для определения места хранения товаров, управления движением между различными складами и оптимизации логистических процессов.

Основные данные\
На этом этапе вводится общая информация о складе:

* **Название** — вводится название склада
* **Тип** — выбирается тип склада (например: основной, дополнительный и другие)
* **Ответственное лицо** — ответственный сотрудник за склад
* **Код** — уникальный код склада (необязательно)
* **Порядковый номер** — номер для сортировки внутри системы
* **Примечание** — дополнительная информация (необязательно)
* **Статус** — активен или неактивен склад
*   **Основной** — отметка как основного склада



Форма и размеры\
Здесь можно ввести физические параметры склада:

* **Форма** — форма склада (прямоугольная, круглая или другая)
* **Высота (м)** — высота склада
* **Ширина (м)** — ширина склада
* **Длина (м)** — длина склада
* **Объём (м³)** — объём склада (автоматически или вручную)

Условия хранения\
Важные параметры для хранения товаров:

* **Рекомендуемая температура (°C)** — оптимальная температура
* **Рекомендуемая влажность (%)** — оптимальный уровень влажности

Местоположение\
Географические и адресные данные склада:

* **GPS координаты** — значения широты и долготы
* **Адрес** — местоположение склада

Значение складов в системе:

* ведение остатков товаров в разрезе складов
* возможность работы с несколькими складами
* управление процессами внутренних и внешних перемещений
* оптимизация логистики и процессов снабжения

<figure><img src="../../.gitbook/assets/image (396).png" alt=""><figcaption></figcaption></figure>

Например:\
Если в компании есть «Большой склад» и «Склад магазина», чётко видно, где находятся товары, и при необходимости осуществляется перемещение между ними.

</details>

<details>

<summary>Типы складов</summary>

Через раздел «Типы складов» можно разделить склады в системе на категории и управлять ими упорядоченно. Этот раздел позволяет группировать склады по их функции и в дальнейшем легко ими управлять.

Для создания типа склада пользователь переходит в раздел **Склад → Типы складов** и нажимает кнопку «Создать».

<figure><img src="../../.gitbook/assets/image (397).png" alt=""><figcaption></figcaption></figure>

Вводятся следующие данные:

* **Название** — название типа склада (например: Основной, Склад магазина, Резервный склад)
* **Порядковый номер** — для определения порядка отображения в системе
* **Статус** — активный или неактивный статус

<figure><img src="../../.gitbook/assets/image (398).png" alt=""><figcaption></figcaption></figure>

Если в системе имеется несколько складов, разделив их на типы, такие как «Основной», «Торговый склад», «Резервный склад», задача каждого из них чётко определяется, и управление становится намного удобнее.

</details>
