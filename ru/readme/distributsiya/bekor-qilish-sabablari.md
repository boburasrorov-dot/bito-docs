# Причины отмены



### Создание причин отмены

Раздел причин отмены позволяет заранее определить в системе причины отмены визитов или операций. Через этот раздел пользователь стандартизирует процесс отмены и формирует базу точных причин для последующего анализа. Этот раздел важен для процессов контроля, отчетности и анализа.

Осуществляется переход в раздел Визиты → Причины отмены. В этом разделе все созданные причины отображаются в виде таблицы.

Столбцы таблицы:

* **№** – порядковый номер
* **Название** – название причины отмены
* **Код** – уникальный код для причины
* **Создавший** – пользователь, создавший причину
* **Тип возврата** – тип отмены (например, действительный)
* **Примечание** – дополнительная информация
* **Активность** – активное или неактивное состояние причины

Для создания новой причины нажимается кнопка "Создать". В открывшемся окне доступны следующие поля:

* **Название** – название причины отмены
* **Код** – идентификатор для причины (необязательно)
* **Тип возврата** – тип отмены (например, действительный или другие состояния)
* **Примечание** – дополнительное пояснение
* **Активность** – указывает, активна причина или нет

Также доступны следующие кнопки управления:

* **Отменить** – отмена действия
* **Сохранить** – сохранение данных

<figure><img src="../../.gitbook/assets/image (509).png" alt=""><figcaption></figcaption></figure>

Основная особенность этого раздела заключается в том, что он позволяет систематически собирать причины отмененных визитов или операций. Это дает пользователю важные данные для определения наиболее часто встречающихся причин, анализа проблем и улучшения процессов.
