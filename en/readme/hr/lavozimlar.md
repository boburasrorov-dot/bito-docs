# Positions



### Positions

**The Positions section** is used to define employees' roles in the system and link them to a specific department. Through this section, various positions within the company (for example: Salesperson, Accountant, Manager, etc.) are created, and employees' responsibilities and their place in the system are determined through them. This greatly simplifies the management of permissions and roles.

Navigate to **HR → Positions** section and add a new position via the **"Create"** button.

In this section, all positions are displayed in table view.

**Table columns:**

* **Name** – position name
* **Number of employees** – number of employees assigned to this position

When creating a new position, the following information is entered:

* **Name** – position name (required)
* **Department** – which department the position belongs to

Additionally, when creating a position, existing employees can be assigned to this position:

* Select the required ones from the employee list
* Use the toggle to assign or remove an employee from this position

Through this:

* multiple employees can be assigned to each position
* employees' responsibilities are clearly defined

The Positions section is **directly linked to permissions**, and later each position can be granted separate permissions. Therefore:

* employees in the same position will have the same rights
* role management in the system is centralized

As a result, a clear structure is formed in the company, and employees' responsibilities and system access rights are managed in an organized manner.
