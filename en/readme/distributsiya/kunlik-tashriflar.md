# Daily Visits

<details open>

<summary>Daily Visits</summary>

The Daily Visits section allows you to view visits scheduled to be performed by agents for a selected date. Through this section, the user can review which agent should visit which customers on a specific day, monitor them, and effectively manage the daily workflow. This section serves as an important tool for operational control and daily planning.

Access the Visits → Daily Visits section. At the top, there is a date filter, allowing the user to select the required date and view visits specifically for that day.

In this section, all visits are displayed in table format.

Table columns:

* **№** – serial number
* **Nomi** – agent (employee) name
* **Mijozlar soni** – number of customers to be visited on this day
* **Buyurtma** – orders related to the visit (if any)
* **Harakatlar** – actions (for example, "Plan qo'yish")

When clicking on each agent, their detailed information opens. This window displays the customers assigned to the agent and their visit status.

<figure><img src="../../.gitbook/assets/image (497).png" alt=""><figcaption></figcaption></figure>

Table columns in the detail window:

* **№** – serial number
* **Nomi** – customer name
* **Turi** – customer type (for example, individual)
* **Reja** – visit type (for example, automatic)
* **Navbat** – visit sequence or status

The main feature of this section is that it works within a single date range and displays only the visits that need to be performed on that specific day. This allows the user to review only the important daily tasks without unnecessary information.

<figure><img src="../../.gitbook/assets/image (498).png" alt=""><figcaption></figcaption></figure>

As a result, control over agent activities is strengthened, visits are performed on time, and the customer interaction process becomes more systematic and efficient.
