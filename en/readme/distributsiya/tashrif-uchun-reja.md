# Visit Plan



### Creating a Visit Plan

The Visit Plan section allows for advance planning of visits to be made by employees (agents) to customers. Through this section, the user can specify which agent will visit which customers, at what frequency, and manage them systematically. This section is used to optimize the sales process, organize customer relations, and monitor agent activities.

After entering the Visit Plan section, a new plan is created through the "**Create**" button. All plans are displayed in table view in this window.

Table columns:

* **№** – serial number
* **Full Name** – agent (employee) name
* **Number of Customers** – number of assigned customers
* **Actions** – operations to be performed on the plan (for example, "Set Plan")

The process of creating a new plan consists of several steps:

In the first step, an employee (agent) is selected. Here, only users who have been assigned the "Agent" and "Courier" roles by HR in the permissions section appear in the list. For this reason, not just any employee, but only those with appropriate permissions are selected as agents.

<figure><img src="../../.gitbook/assets/image (495).png" alt=""><figcaption></figcaption></figure>

In the second step, customers to be assigned to the agent are selected. In the "Customers" section, the list of customers to be visited by the selected agent is formed. If customers have not yet been added, the table appears empty (showing "No data found"). Additionally, there is an "Auto-fill" function available for quick customer selection.

<figure><img src="../../.gitbook/assets/image (496).png" alt=""><figcaption></figcaption></figure>

In the third step, visit date and recurrence parameters are set.\
Start date – indicates from which date the plan begins\
Recurrence – visit frequency (daily, weekly, monthly, etc.)\
Number of recurrences – indicates how many visits will be made in one day or during the selected period

For example, if "daily" and "2" are specified, this means that 2 visits per day are scheduled for the customer.

Additional control buttons are also available in this section:

* **Cancel** – cancels the plan being created
* **Next** – proceed to the next step
* **Save** – save plan data

The main feature of this section is that it allows for clear and systematic planning of the visit process between agents and customers. This increases agent work efficiency, ensures regular work with customers, and makes the sales process more organized.
