# Visit Plan

<details open>

<summary>Creating a Visit Plan</summary>

The visit plan section allows for advance planning of visits to be made by employees (agents) to customers. Through this section, users can specify which agent will visit which customers, at what frequency, and manage them systematically. This section is used to optimize the sales process, organize customer relations, and monitor agent activities.

After entering the visit plan section, a new plan is created through the "**Create**" button. In this window, all plans are displayed in table view.

Table columns:

* **№** – serial number
* **Full Name** – agent (employee) name
* **Number of Customers** – number of assigned customers
* **Actions** – operations performed on the plan (for example, "Set Plan")

The process of creating a new plan consists of several stages:

In the first stage, an employee (agent) is selected. Here, only users who have been assigned the "Agent" role by HR in the permissions section appear in the list. Therefore, not just any employee, but only those with the appropriate permissions are selected as agents.

<figure><img src="../../.gitbook/assets/image (495).png" alt=""><figcaption></figcaption></figure>

In the second stage, customers to be assigned to the agent are selected. In the "Customers" section, a list of customers to be visited is formed for the selected agent. If customers have not yet been added, the table appears empty ("No data found"). Additionally, there is an "Auto-fill" feature for quick customer selection.

<figure><img src="../../.gitbook/assets/image (496).png" alt=""><figcaption></figcaption></figure>

In the third stage, the visit date and recurrence parameters are set.\
Start date – indicates from which date the plan begins\
Recurrence – visit frequency (daily, weekly, monthly, etc.)\
Number of recurrences – indicates how many visits will be made in one day or within the selected period

For example, if "daily" and "2" are specified, this means 2 visits per day are scheduled for the customer.

Additional control buttons are also available in this section:

* **Cancel** – cancels the plan being created
* **Next** – proceed to the next stage
* **Save** – save plan data

The main feature of this section is that it enables precise and systematic planning of the visit process between agents and customers. This increases agent work efficiency, ensures regular customer engagement, and makes the sales process more organized.
