# Cancellation Reasons



### Creating Cancellation Reasons

The cancellation reasons section allows you to predefine reasons in the system when visits or operations are cancelled. Through this section, the user standardizes the cancellation process and forms a clear database of reasons for later analysis. This section is important for control, reporting, and analysis processes.

Navigate to Visits → Cancellation Reasons section. In this section, all created reasons are displayed in table view.

Table columns:

* **№** – sequential number
* **Name** – cancellation reason name
* **Code** – unique code for the reason
* **Creator** – user who created the reason
* **Return Type** – cancellation type (for example, valid)
* **Comment** – additional information
* **Active** – active or inactive status of the reason

To create a new reason, click the "Create" button. The opened window contains the following fields:

* **Name** – cancellation reason name
* **Code** – identifier for the reason (optional)
* **Return Type** – cancellation type (for example, valid or other statuses)
* **Comment** – additional explanation
* **Active** – indicates whether the reason is active or not

Additionally, the following control buttons are available:

* **Cancel** – cancel the action
* **Save** – save the data

<figure><img src="../../.gitbook/assets/image (509).png" alt=""><figcaption></figcaption></figure>

The main feature of this section is that it allows systematic collection of reasons for cancelled visits or operations. This provides the user with important data to identify which reasons occur most frequently, analyze problems, and improve processes.
