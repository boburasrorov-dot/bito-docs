# Photo Report Type

<details open>

<summary>Photo Report Type</summary>

The Photo Report Type section allows you to create and manage types (categories) for photo reports submitted by agents. Through this section, the user pre-defines what types of photos are required from agents. For example, types such as "Product Photo" or "Store Photo" are created, and the agent is required to upload exactly these types of photos during the visit process. This section is used to strengthen control and improve visit quality.

Access is made to **Visits → Photo** Report Type section. In this section, all created photo report types are displayed in table view.

Table columns:

* **№** – serial number
* **Full Name** – photo report type name

<figure><img src="../../.gitbook/assets/image (503).png" alt=""><figcaption></figcaption></figure>

To create a new photo report type, the "Create" button is pressed. In the opened window, the following fields are available:

* **Name** – photo report type name (for example, "Product Photo", "Store Photo")
* **Mandatory** – specifies whether uploading this type of photo is mandatory or not
* **Active** – specifies that the type is in active status

Also, the following control buttons are available:

* **Delete** – delete existing type
* **Cancel** – cancel action
* **Save** – save data

The main feature of this section is that it sets standard requirements for agents. That is, if several photo report types have been created, the agent will need to upload photos for all these types during the visit (if "Mandatory" is specified).

As a result, the company establishes visual control in the visit process, creates the opportunity to check agents' work more accurately, and report quality improves significantly.
