<summary>Visits</summary>

The Visits section allows you to view all completed visits carried out in the system in one place. Through this section, the user can review the history of visits performed by agents, analyze sales results created based on them, and assess overall performance efficiency. This section is of great importance for reporting and analysis processes.

<figure><img src="../../.gitbook/assets/image (502).png" alt=""><figcaption></figcaption></figure>

Access **Visits → Visits** section. In this section, all visits are displayed in table view and the user has the ability to search, filter by date or employee.

Table columns:

* **№** – sequence number
* **Number** – unique identifier of the visit
* **Organization** – organization where the visit was carried out (branch or company)
* **Date** – date when the visit was carried out
* **Total price** – sales amount made during the visit
* **Client** – name of the visited client
* **Agent** – employee who carried out the visit

Additional features are also available in this section:

* **Search** – for quick finding of visits
* **Filter (Date, Employee)** – for sorting data by required parameters
* **Export** – for downloading the complete list of visits

The main feature of this section is that it shows only completed visits. That is, visits that have been performed and registered in the system are stored here. This provides the user with accurate data for analyzing past activities, evaluating agent performance, and planning for the future.
