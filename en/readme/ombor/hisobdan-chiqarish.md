# Write-off

<details open>

<summary>Write-off </summary>

The write-offs section is used to carry out the process of writing off products from the warehouse based on various reasons. This process is used to formalize the removal of products from the warehouse due to expiration, loss, or other reasons.

The user goes to **Warehouse → Write-offs** section and clicks the **"Create"** button to create a new document.

<figure><img src="../../.gitbook/assets/image (369).png" alt=""><figcaption></figcaption></figure>

When creating a write-off, the following information is entered:

* **WO number** — document number
* **Date** — write-off date
* **Warehouse** — warehouse from which the product is being removed
* **Responsible person** — responsible employee
* **Reason** — pre-created write-off reason
* **Currency** — accounting currency
* **Action** — write-off type\
  (for example: mark as defective and others)
* **Defect warehouse** — if the product is classified as defective, which warehouse it will be transferred to
* **Note** — additional information (optional)
* **Upload file** — attach document (optional)

At the bottom, products to be written off are selected.

The user:

* searches for or selects products
* enters the quantity to be removed
* views the available quantity in the warehouse

<figure><img src="../../.gitbook/assets/image (370).png" alt=""><figcaption></figcaption></figure>

For each product:

* **Quantity in warehouse** — current balance
* **Quantity** — quantity to be written off
* **Box quantity** — quantity by packaging
* **Expiration date** — if available
