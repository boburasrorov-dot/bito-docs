# Categories

<details open>

<summary>Categories</summary>

The Categories section organizes products and services and provides the ability to group them. This section ensures structured data management and efficient operation in the system.

To create a category, the user navigates from the left menu to **Warehouse → Categories** section and clicks the "Create" button.

<figure><img src="../../.gitbook/assets/image (393).png" alt=""><figcaption></figcaption></figure>

The main purpose of the Categories section is to divide products and services into logical groups. For example: categories such as "Beverages", "Electronics", "Services" create order and convenient navigation in the system.

<figure><img src="../../.gitbook/assets/image (394).png" alt=""><figcaption></figcaption></figure>

Basic Information\
At this stage, general information about the category is entered:

* **Name** — the category name is entered
* **Parent Category** — if the category is a subcategory, its parent category is selected
* **Available in Sales** — enable or disable the category for use in sales

Organizations\
Here you specify in which organizations the category will be active:

* **Organizations List** — select in which branch or company the category will be used
* **Activity Status** — the category is enabled or disabled for each organization

The importance of categories in the system:

* provides the ability to manage products and services in an organized manner
* speeds up search and filtering processes
* helps with precise segmentation in reports
* enables the creation of hierarchy through subcategory structure

Example:\
If a "Electronics" category is created, subcategories such as "Phones", "Laptops" can be opened within it. This allows products to be managed more precisely and conveniently.
