# Transfer Reasons

<details open>

<summary>Creating transfer reasons</summary>

Through the Transfer Reasons section, it is possible to organize and control product transfer processes between warehouses based on reasons. This section provides a detailed and systematic approach to each transfer operation.

To create a transfer reason, the user navigates to **Warehouse → Transfer Reasons** section and clicks the "Create" button.

<figure><img src="../../.gitbook/assets/image (399).png" alt=""><figcaption></figcaption></figure>

The following information is entered:

* **Name** — transfer reason name (e.g.: Inter-warehouse transfer, Delivery, Transfer to reserve)
* **Code** — short identifier for the reason (optional)
* **Sort order** — determines the display order in the list
* **Description** — additional explanation (optional)
* **Active** — indicates whether the reason is active or inactive

<figure><img src="../../.gitbook/assets/image (401).png" alt=""><figcaption></figcaption></figure>

For example:\
If a product is transferred from one warehouse to another with the reason "Transfer to reserve", later in reports it will be clearly visible exactly how much product was transferred for which reason.
