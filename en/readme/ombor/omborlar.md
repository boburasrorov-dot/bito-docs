# Warehouses

<details>

<summary>Warehouses</summary>

Through the Warehouses section, locations where products are stored (warehouses) are created and managed in the system. This section enables proper organization of the warehouse structure and precise control over product movement.

To create a warehouse, the user navigates from the left menu to **Warehouse → Warehouses** and clicks the "Create" button.

<figure><img src="../../.gitbook/assets/image (395).png" alt=""><figcaption></figcaption></figure>

The Warehouses section serves to identify where products are stored, manage movement between different warehouses, and optimize logistics processes.

Basic Information\
At this stage, general information about the warehouse is entered:

* **Name** — warehouse name is entered
* **Type** — warehouse type is selected (for example: main, additional, and others)
* **Responsible Person** — the employee responsible for the warehouse
* **Code** — unique code for the warehouse (optional)
* **Sequence Number** — number for ordering within the system
* **Note** — additional information (optional)
* **Status** — whether the warehouse is active or inactive
*   **Main** — designation as the main warehouse



Shape and Dimensions\
Here the warehouse's physical parameters can be entered:

* **Shape** — warehouse shape (rectangular, circular, or other)
* **Height (m)** — warehouse height
* **Width (m)** — warehouse width
* **Length (m)** — warehouse length
* **Volume (m³)** — warehouse volume (automatic or manual)

Storage Conditions\
Important parameters for storing products:

* **Recommended Temperature (°C)** — optimal temperature
* **Recommended Humidity (%)** — optimal humidity level

Location\
Geographical and address information of the warehouse:

* **GPS Coordinates** — latitude and longitude values
* **Address** — warehouse location

Importance of warehouses in the system:

* maintaining product balances by warehouse
* ability to work with multiple warehouses
* managing internal and external transfer processes
* optimizing logistics and supply processes

<figure><img src="../../.gitbook/assets/image (396).png" alt=""><figcaption></figcaption></figure>

For example:\
If a company has a "Main Warehouse" and "Store Warehouse," it is clearly visible where products are located and transfers can be made between them when needed.

</details>

<details>

<summary>Warehouse Types</summary>

Through the Warehouse Types section, warehouses can be categorized in the system and managed in an organized manner. This section enables grouping warehouses according to their function and facilitates their management later.

To create a warehouse type, the user navigates to **Warehouse → Warehouse Types** and clicks the "Create" button.

<figure><img src="../../.gitbook/assets/image (397).png" alt=""><figcaption></figcaption></figure>

The following information is entered:

* **Name** — warehouse type name (for example: Main, Store Warehouse, Reserve Warehouse)
* **Sequence Number** — to set the display order within the system
* **Status** — active or inactive status

<figure><img src="../../.gitbook/assets/image (398).png" alt=""><figcaption></figcaption></figure>

If there are multiple warehouses in the system, dividing them into types such as "Main," "Sales Warehouse," "Reserve Warehouse" clearly defines the function of each and makes management much more convenient.

</details>
