# Inventory

<details open>

<summary>Inventory</summary>

Through the Inventory section, the process of comparing warehouse product balances with the actual state and identifying discrepancies is carried out. This process ensures complete control over warehouse products and allows for the detection of accounting errors.

The user goes to **Warehouse → Inventory** section and clicks the **"Create"** button.

<figure><img src="../../.gitbook/assets/image (360).png" alt=""><figcaption></figcaption></figure>

#### **Basic Information**

When creating an inventory, the following information is entered:

<figure><img src="../../.gitbook/assets/image (361).png" alt=""><figcaption></figcaption></figure>

* **Responsible person** — employee responsible for the inventory
* **Warehouse** — warehouse to be inventoried
* **Start date** — process start time
* **End date** — process completion time
* **Comment** — additional information (optional)
* **Upload file** — attach document (optional)

Additional settings:

* **Full** — all products in the warehouse are automatically added to the inventory
* **Partial** — only selected products are added to the inventory

When "**Create with warehouse quantity**" is selected, the system generates products with their current balance quantity.



#### **Products**

If **Partial** is selected, the user manually selects the products to be inventoried in the lower window.

If **Full** is selected, all products are automatically added to the list.

For each product:

**Warehouse quantity** — current balance in the system\
**Counted quantity** — actual quantity to be counted and entered\
**Difference** — calculated automatically

<figure><img src="../../.gitbook/assets/image (364).png" alt=""><figcaption></figcaption></figure>

#### **Process**

After the user enters all information:

**Save** — inventory document is created and opened for subsequent counting

**Save and count** — document is saved and user immediately proceeds to the counting window

During the counting process, the user enters the actual quantity of products.
