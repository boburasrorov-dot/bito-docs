# Intercompany Transfers



### Intercompany Transfer Requests

The intercompany transfer requests section is used to create advance requests for product transfers between organizations. This stage represents the agreement and planning process before the actual transfer.

The user enters the **Warehouse → Intercompany Transfer Requests** section and creates a new request through the "Create" button.

<figure><img src="../../.gitbook/assets/image (407).png" alt=""><figcaption></figcaption></figure>

The following information is entered:

* **Sending Organization** — the organization sending the product
* **Status** — request status (e.g., New, Approved)
* **CTR Number** — document number
* **Time** — request creation time
* **Transfer Time** — scheduled date
* **Sender** — responsible employee
* **Reason** — reason for transfer
* **File Upload** — attach document (optional)
* **Comment** — additional information

Products are added in the lower section:

* product is selected
* required quantity is entered
* total quantity is calculated automatically

<figure><img src="../../.gitbook/assets/image (408).png" alt=""><figcaption></figcaption></figure>

Additional features:

* **Import** — upload products via file
* **Quick Select** — fast addition
* **Auto-fill** — automatic quantity entry

Through this section, product movement between organizations is planned and controlled in advance.







### Intercompany Transfers

The intercompany transfers section is where the actual transfer of products between organizations takes place. This document is typically created based on an intercompany transfer request.

The user enters the **Warehouse → Intercompany Transfers** section and clicks the "Create" button.

<figure><img src="../../.gitbook/assets/image (409).png" alt=""><figcaption></figcaption></figure>

The following information is entered:

* **Receiving Organization** — the organization receiving the product
* **Status** — document status
* **CT Number** — document number
* **Time** — operation time
* **Shipment Date** — shipping date
* **Sending Warehouse** — warehouse from which product is dispatched
* **Sender** — responsible employee
* **Mark as Loan** — checked if product is given as a loan
* **Reason** — reason for transfer
* **File Upload** — attach document
* **Comment** — additional comment

In the products section:

* product is selected
* available quantity in warehouse is displayed
* quantity at receiving organization is also shown
* transfer quantity is entered
* expiration date is selected if necessary

<figure><img src="../../.gitbook/assets/image (410).png" alt=""><figcaption></figcaption></figure>

Additional features:

* **Import** — upload products
* **Quick Select** — fast addition
* **Auto-fill** — fill based on balance

Through this section:

* product movement between organizations is managed
* balances are automatically updated on both sides
* loan-based transfers are processed
* logistics and accounting processes are simplified

For example:\
If one organization sends products to another branch, the process is formalized through an intercompany transfer document, and product balances change accordingly in both organizations.
