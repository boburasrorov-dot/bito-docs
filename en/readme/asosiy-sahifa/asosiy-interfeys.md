### Side Menu (Navigation Panel)

The side menu is used to navigate between system modules.

Each module manages a specific business process:

* **Reports** – view analytical and statistical data
* **Sales** – sales documents and operations
* **CRM** – customer relationship management
* **Supply** – purchasing and delivery
* **Warehouse** – product movement
* **Finance** – cash flow and accounting
* **Production** – manufacturing process
* **Settings** – system configuration

Purpose: to manage business processes separated by modules

<div><figure><img src="../../.gitbook/assets/image (196).png" alt=""><figcaption></figcaption></figure> <figure><img src="../../.gitbook/assets/image (197).png" alt=""><figcaption></figcaption></figure></div>

Each module also has its own sub-modules, meaning actions that are part of that module. For example:

* Sales - section for creating new sales and viewing all sales
* Label - section for printing labels
* Returns - section for returning sold products and viewing returned products
* Installment - section for controlling installment payments
* Plan Setting - section for setting sales plans and tracking them
* Sales Dashboard - general sales-related dashboard
* Customer Payments - section for managing and monitoring debtor customer payments

### Filtering

The filtering function in BITO helps find the needed data quickly and accurately. Each table page has filters available, allowing users to sort data by date, status, responsible person, warehouse, customer, and other columns.

BITO has 3 types of filter views:

* Inline
* Static
* Collapsible

### Configuring Filter View

Users can make the filter view the same across all pages:

**Settings → User Settings → Filter → Select**

Here, users can select their preferred filter type (inline, static, or expandable) and set it as the standard view throughout the system.
