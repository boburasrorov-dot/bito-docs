# Additional Prices

<details open>

<summary>Additional prices (expenses)</summary>

The Additional Prices section is used to create and manage additional expenses that arise during sales or procurement processes. This section enables you to link logistics, service, tax, or other additional expenses to product prices and calculate them correctly.

Users can view existing additional prices, edit them, or add new additional expenses here.

#### Creating an additional price

A new additional price is added via the **"Create"** button, and the following information is entered:

* **Name** — name of the additional expense (e.g., delivery, customs, advertising, etc.)
* **Suggested price** — default value, can be applied automatically
* **Maximum price** — maximum allowed value for this expense
* **Currency** — currency in which the expense is calculated

<figure><img src="../../.gitbook/assets/image (454).png" alt=""><figcaption></figcaption></figure>

* **Payment type** — indicates what type of expense this belongs to\
  (e.g., office services, taxes, rent, communication expenses, etc.)
* This field is linked to transaction types and is correctly reflected in financial accounting

<figure><img src="../../.gitbook/assets/image (455).png" alt=""><figcaption></figcaption></figure>

* **Status** — specifies whether the additional price is active or inactive
* **Affects price** — if enabled, this expense is added to the product's cost or sales price
* **Distribution method** — specifies how the expense is distributed across products:
  * **By quantity** — distributed based on product quantity
  * **By price** — distributed proportionally to product value
  * **By volume (m³)** — distributed based on volume
  * **By gross weight (kg)** — distributed based on weight
  * **Manual** — manually distributed by the user
* **Comment** — field for entering additional information

<figure><img src="../../.gitbook/assets/image (457).png" alt=""><figcaption></figcaption></figure>

Once an additional expense is created, it is applied in sales or procurement documents. If the **"Affects price"** function is enabled, this expense is added to the product's cost or affects the sales price.

For example, if a delivery expense is added and set to affect the price, the final price of the product automatically increases or is added to the cost.
