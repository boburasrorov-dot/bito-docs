# Currencies

<details open>

<summary>Currencies</summary>

The Currencies section allows you to create and manage all monetary units used in the system. This section enables working with different currencies, performing conversions, and properly conducting financial operations.

To add a currency, the user navigates to **Finance → Currencies** from the left menu and clicks the **"Create"** button.

The Currencies section serves for maintaining multi-currency accounting, setting exchange rates, and managing payments in different monetary units.

#### Main fields and their functions

* **Global equivalent** — the international standard of the currency is selected. This allows the system to align the currency with global monetary units and provides a foundation for future integrations.
* **Name** — the currency name displayed within the system. This exact name is used in reports, transactions, and all financial windows.
* **Symbol** — the currency symbol ($, UZS, etc.). This allows users to view amounts quickly and clearly.
* **Position** — determines where the currency symbol appears relative to the number (before or after). This is important for UI/UX and adapts to different country standards.
* **Base currency** — designates this currency as the system's primary (base) currency. All other currencies are converted relative to this currency.
* **Precision (number of decimal places)** — defines how many decimal places to use in calculations. For example, if 2: 100.25, if 0: 100.
