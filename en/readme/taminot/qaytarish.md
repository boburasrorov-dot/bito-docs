# Returns

<details open>

<summary>Returns</summary>

Through the Supply → Returns section, the process of returning products for purchases made in the system is carried out. In this section, returns are performed simply and quickly — all data is entered in one window without being divided into separate sections.

<figure><img src="../../.gitbook/assets/image (61).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (78).png" alt=""><figcaption></figcaption></figure>

The return process is carried out in two ways:

#### **Creating through the Returns section**

The user enters the Supply → Returns section and creates a new return document.

The following information is entered in this window:

• **Warehouse** — select which warehouse the returned products will be received into.\
• **Purchase** — select which purchase document the return is based on (optional).\
• **Supplier** — select the supplier to whom the product is being returned.\
• **SRN number** — additional number related to the return document (optional).\
• **Time** — set the date and time when the return is being carried out.\
• **Reason** — select the reason for the return (mandatory).\
• **Responsible person** — designate the employee responsible for the return.\
• **Currency** — select the currency in which the return calculation will be performed.\
• **Upload file** — attach necessary documents.\
• **Comment** — write additional information.

<figure><img src="../../.gitbook/assets/image (80).png" alt=""><figcaption></figcaption></figure>

Then in the lower section, the user selects the products to be returned:\
• add the necessary products\
• enter the quantity to be returned

The system automatically calculates the total amount based on the entered data.

After all information is filled in, the document is saved and the return is carried out.

#### **Return from within the purchase document**

The second method of return — is carried out from within an existing purchase document.

For this, the user:

* opens the necessary purchase document
* presses the **"Return"** button located on the right side

<figure><img src="../../.gitbook/assets/image (81).png" alt=""><figcaption></figcaption></figure>

As a result, the return window opens and some information is automatically filled in (supplier, products, and others).

The user:\
• selects the return reason\
• enters the quantity of necessary products\
• enters additional information if needed

<figure><img src="../../.gitbook/assets/image (82).png" alt=""><figcaption></figcaption></figure>

Then the return document is saved.

As a result, the return process is carried out simply and quickly, products are returned to the warehouse, and all operations are fully reflected in the system.
