# Categories

<details open>

<summary>Creating a Category</summary>

In the Supply → Categories section, clicking the **Create** button opens a new category creation window.

<figure><img src="../../.gitbook/assets/image (63).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (48).png" alt=""><figcaption></figcaption></figure>

Through this section, suppliers can be divided into different categories. Categories enable systematic management of suppliers, grouping them, and subsequently allowing for quick filtering and analysis.

When creating a category, the following information is entered:

<figure><img src="../../.gitbook/assets/image (49).png" alt=""><figcaption></figcaption></figure>

• **Name** — the category name is entered.\
• **Parent Category** — if necessary, a parent category is selected for the category. This allows for creating a hierarchical structure.

Through the following section, it is specified which objects the category belongs to:

• A list of existing organizations or departments in the system is displayed\
• The category is assigned by toggling on/off for each element

After all information is entered, it is saved and the category is added to the system.\
As a result, the ability to group suppliers is created, and working with them becomes more organized and convenient.
