# Return Reasons

<details open>

<summary>Creating Return Reasons</summary>

In the Supply → Return Reasons section, when the **Create** button is pressed, a new return reason creation window opens.

<figure><img src="../../.gitbook/assets/image (67).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (53).png" alt=""><figcaption></figcaption></figure>

Through this section, it is possible to predefine reasons for product returns. This is crucial for organizing return processes, identifying issues, and conducting analysis.

When creating a return reason, the following information is entered:

• **Name** — the name of the return reason is entered (for example: defect, wrong product, quality issue, etc.).\
• **Code** — a special code for the reason is entered (optional).\
• **Sequence Number** — determines the display order in the reasons list.\
• **Comment** — additional information or explanation is written.\
• **Active** — indicates whether the reason is active or inactive.

<figure><img src="../../.gitbook/assets/image (54).png" alt=""><figcaption></figcaption></figure>

After all information is entered, it is saved and the reason is added to the system.

As a result, return operations are conducted based on specific reasons, which helps quickly identify issues, perform statistical analysis, and reduce errors in the future.

</details>
