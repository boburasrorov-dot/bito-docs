# Prices

<details open>

<summary>Creating prices</summary>

The **Prices** section allows you to create various price types for products and manage them in a centralized manner. Through this section, users can set prices according to different sales strategies, apply them to products, and control the overall pricing policy. Additionally, there is functionality for automatic price calculation and management through limits (minimum/maximum). This section is used to implement pricing policy and improve sales efficiency.

<figure><img src="../../.gitbook/assets/image (2).png" alt=""><figcaption></figcaption></figure>

Navigate to Data → Prices section and create a new price type using the **"Create"** button.

All prices are displayed in table format in this section.

Table columns:

* **Name** – price type name (for example: retail, wholesale)
* **Code** – short code for the price
* **Type** – price type (Sales or Purchase)
* **Created by** – user who created the price
* **Created at** – date and time when the price was created
* **Modified by** – user who made the last modification
* **Modified at** – date and time of the last update
* **Actions** – "Pricing" button to apply the price to products

<figure><img src="../../.gitbook/assets/image (4).png" alt=""><figcaption></figcaption></figure>

Created prices are attached to products through the **"Pricing"** button on the right side. In this window, prices can be entered manually or automatically (based on percentage or amount) for products.

This section serves to manage overall prices, while within a product, only the price related to that product is displayed. Therefore, users can manage prices for all products in a single location rather than individually. This enables quick price updates, reduces errors, and allows for optimization of the sales process.
