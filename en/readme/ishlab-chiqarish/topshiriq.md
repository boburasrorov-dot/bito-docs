# Task

<details open>

<summary>Creating a Task</summary>

The Task section is used to launch and manage the production process in practice. Here, production tasks (assignments) are created based on previously created technical cards, and the workflow is monitored.

Navigate to Production → **Task** section and create a new task.

<figure><img src="../../.gitbook/assets/image (20).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (28).png" alt=""><figcaption></figcaption></figure>

When creating a task, the following main information is entered:

* **Responsible person**\* – the employee responsible for the task
* **Production warehouse**\* – the warehouse where the process takes place
* **Currency**\* – currency for calculations
* **Code** – task identifier
* **Comment** – additional information



The most important part of the task is where the **technical card is selected**.

* When a **Technical card** is selected, the system automatically loads the **stages** associated with it
* Each stage represents a sequential step in the production process
* **Product** and **quantity** are entered
* **Norm** – standard production indicator
* **Planned** – the volume planned for execution
