# Tasks

<details open>

<summary>Working with Tasks</summary>

When entering **CRM → Tasks**, a list of all tasks created in the system is displayed. Tasks can be viewed by day, week, and month.

Through this section, you can assign tasks to employees, monitor them, and track their completion status.

Tasks can be linked to sales, customer interactions, or internal processes. This helps maintain organized workflows.

<figure><img src="../.gitbook/assets/image (314).png" alt=""><figcaption></figcaption></figure>

To create a new task, press the **Create** button and enter the following information:

<figure><img src="../.gitbook/assets/image (315).png" alt=""><figcaption></figcaption></figure>

• **Sale** — select which sale or process the task is associated with.

• **Responsible person** — specify the employee who will execute the task.

• **Type** — task type (for example, debt, assignment, and others).

• **Date and time** — set the time period when the task should be completed.

• **Comment** — write additional information about the task.

Tasks are monitored in table or calendar view.

Tasks can be controlled based on their status (for example: completed, not completed, overdue).

As a result, employee activities are monitored through tasks and workflows are managed in an organized manner.
